Application API Documentation

Secure API reference for the UW Madison Data Science for Sustainable Development program application system.

Overview

The DSSD Application API provides secure, email-based access to submit and manage applications for Data Science for Sustainable Development.

Base URL: https://f25-dssdapp.onrender.com

General Club Information

The Data Science for Sustainable Development program offers students the opportunity to:

  • Engage in industry partnerships and collaborative research projects
  • Develop advanced technical skills in software development
  • Build professional networks with peers and industry leaders
  • Work and learn on projects that really matter and make a difference
Ready to apply? Keep reading this documentation to tell you how. For any questions on how we recruit click here! Visit our main website to learn more about the program!

Quick Start Guide

1
Prepare your application data

Gather required information: first name, last name, email address, and expected graduation year (2025-2030).

2
Submit your application

Send a POST request to /apply with your application data in JSON format.

3
Check your email for confirmation

You'll receive an email with your application ID (format: app_xxxxx) for future reference.

4
Save your application ID

Store the application ID from your email to manage your application later.

Security Model

The API uses a simplified security model based on application IDs to ensure data integrity while maintaining ease of use.

Key Security Features

  • Unique Email Constraint: Only one application per email address is allowed
  • Email Confirmation: All operations send confirmation emails to the registered address
  • Simple Deletion: Applications can be deleted using only the application ID
  • No Authentication Required: Public endpoints for both submission and deletion
Rate Limiting: To prevent abuse, we enforce a rate limit of 10 requests per 5-minute window per IP address on the /apply endpoint.
Duplicate Email Protection: If you try to submit an application with an email that already exists, you'll receive an EMAIL_ALREADY_EXISTS error instead of overwriting the existing application.

API Endpoints

GET /health Health Check

Check if the API server is running and healthy. No parameters required.

Example Request

GET /health
Success Response (200 OK)
{ "status": "healthy", "timestamp": "2025-09-20T15:30:45.123Z", "environment": "development" }
POST /apply Submit New Application

Creates a new application for the DSSD program. Each email address can only have one application. This endpoint is rate limited to 10 requests per 5-minute window.

Request Body

Parameter Type Required Description
firstName String Yes Applicant's first name
lastName String Yes Applicant's last name
email String Yes Valid email address (must be unique)
year Number Yes Expected graduation year (2025-2030)

Example Request

{ "firstName": "Jane", "lastName": "Smith", "email": "jane.smith@wisc.edu", "year": 2026 }
Success Response (201 Created)
{ "message": "Application submitted successfully", "applicationId": "app_1234567890abcdef", "email_sent": "Confirmation email sent to jane.smith@wisc.edu" }
Duplicate Email Error (400 Bad Request)
{ "error": "An application with this email already exists.", "errorType": "EMAIL_ALREADY_EXISTS" }
Validation Error (400 Bad Request)
{ "error": "Invalid request data", "details": [ "firstName is a required field", "email must be a valid email" ], "errorType": "INVALID_REQUEST" }
DELETE /apply?applicationId=app_xxxxx Delete Application

Permanently deletes an application from the system using the application ID provided in the confirmation email.

Query Parameters

Parameter Type Required Description
id String Yes Unique application identifier (format: app_xxxxx)
Note: The application ID is sent to your email address when you submit your application. It follows the format app_ followed by a unique identifier.
Warning: This action cannot be undone. The application will be permanently removed from our system.

Example Request

DELETE /apply?id=app_1234567890abcdef
Success Response (200 OK)
{ "message": "Application deleted successfully", "applicationId": "app_1234567890abcdef", "email_sent": "Confirmation email sent to applicant@email.com" }
Invalid ID Error (400 Bad Request)
{ "error": "Invalid or missing application ID", "errorType": "INVALID_APPLICATION_ID" }
Application Not Found Error (404 Not Found)
{ "error": "Application not found", "errorType": "APPLICATION_NOT_FOUND" }

Email Notifications

All successful API operations automatically trigger email notifications to the applicant's registered email address.

Note: Currently, email functionality is integrated but may require additional setup with Resend or another email provider.

Application Submission Email

When you submit an application, you'll receive an email containing:

  • Confirmation of successful submission
  • Your complete application details
  • Your unique application ID (format: app_xxxxx)
  • Instructions on how to delete your application if needed
  • Next steps in the application process
  • Support contact information

Application Deletion Email

When you delete your application, you'll receive a confirmation email with:

  • Confirmation that your application was successfully deleted
  • The application ID that was removed
  • Information about reapplying if desired
Important: Keep your application ID safe. It's the only way to delete your application if you need to withdraw it later.

Error Handling

The API uses conventional HTTP response codes to indicate the success or failure of requests. Error responses include detailed error messages and error types to help diagnose issues.

HTTP Status Codes

400 INVALID_REQUEST

Cause: Request is missing required fields or contains invalid data.

Resolution: Verify all required fields are present and correctly formatted. Ensure email format is valid and year is between 2025-2030.

400 EMAIL_ALREADY_EXISTS

Cause: An application with the provided email address already exists.

Resolution: Each email address can only have one application. Use the deletion endpoint to remove your existing application if you want to submit a new one.

400 INVALID_APPLICATION_ID

Cause: The application ID is missing or doesn't start with app_.

Resolution: Verify the application ID is correct and includes the app_ prefix.

404 APPLICATION_NOT_FOUND

Cause: The specified application ID does not exist in our system.

Resolution: Verify the application ID is correct. Contact support if you've lost your application information.

429 RATE_LIMIT_EXCEEDED

Cause: Too many requests sent within the allowed time window on the /apply endpoint.

Resolution: Wait 5 minutes before retrying. Current limit: 10 requests per 5-minute window.

500 DATABASE_ERROR

Cause: Internal server error occurred while processing the request.

Resolution: Retry the request after a few minutes. If the problem persists, contact our support team.

Error Response Format

{ "error": "Invalid request data", "details": [ "firstName is a required field" ], "errorType": "INVALID_REQUEST" }

Rate Limiting

To ensure fair usage and system stability, the API implements rate limiting on the /apply endpoint based on IP address.

Current Limits:
  • 10 requests per 5-minute window on /apply endpoint
  • Limit resets every 5 minutes
  • Rate limiting is applied per IP address
  • Other endpoints (like /health) are not rate limited

Rate Limit Response

When you exceed the rate limit, you'll receive:

{ "error": "Too many requests, please try again later (like in a min).", "errorType": "RATE_LIMIT_EXCEEDED" }
